## 🛠️ 開發與自訂指南

### 1. 照片路徑集中替換
所有照片路徑已集中管理在 `script.js` 檔案頂部的 `CONFIG` 物件中。你只需要把照片放入對應的資料夾，或者修改該陣列內的路徑即可：

```javascript
const CONFIG = {
    // 只要把相片放入 photos/ 資料夾，並命名為 photo1.jpg 到 photo10.jpg 即可！
    // 或者是使用雲端照片網址 (如 Imgur) 代替 'photos/photo1.jpg' 也是可以的喔！
    photos: [
        { path: 'photos/photo1.jpg', caption: '我們在一起了 ❤️' },
        { path: 'photos/photo2.jpg', caption: '第一次吵架又和好 🩹' },
        ...
    ]
}
```

*註：如果照片還沒準備好，網頁有寫防呆機制 (`onerror`)，會自動顯示漂亮的粉紅漸層預設圖。*

### 2. 背景音樂替換
目前 index.html 預留的 `<audio>` 標籤指向 `music/slowly_love_you.mp3`。
請將你下載的《慢慢喜歡你》mp3 檔案放入專案目錄下的 `music` 資料夾，並命名為 `slowly_love_you.mp3` 即可。

## 🚀 部署到 GitHub Pages

這是一個純靜態網頁（HTML+CSS+JS），非常適合免費部署至 GitHub Pages：

1. **建立 GitHub 儲存庫 (Repository)**：
   - 登入 GitHub，建立一個名為 `anniversary` 的公開 (Public) 儲存庫。
2. **上傳程式碼**：
   - 將本專案的所有檔案 (`index.html`、`style.css`、`script.js`、`README.md` 以及 `photos/`、`music/` 資料夾）上傳至該儲存庫的主分支 (main)。
3. **開啟 GitHub Pages 服務**：
   - 在該儲存庫的頁面，點選頂部選單的 **Settings** (設定)。
   - 在左側選單找到 **Pages**。
   - 在 **Build and deployment** 下的 Source 選擇 `Deploy from a branch`。
   - Branch 選擇 `main` 分支並選擇 `/ (root)`，點選 **Save** 儲存。
4. **預覽網站**：
   - 等待約 1 到 2 分鐘，GitHub 會提供一個專屬網址，如：`https://你的帳號.github.io/anniversary/`。
   - 用手機打開這個網址，就可以給寶寶一個滿滿的驚喜囉！🎉
